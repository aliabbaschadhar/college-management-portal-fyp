# Agent Code Generation Rules

You are an expert Next.js, TypeScript, and Prisma ORM engineer. When generating code for this project, you must strictly follow these technical boundaries. 

## 1. Next.js App Router Strictness
* **Unified Stack:** We use Next.js exclusively for both frontend (React) and backend API (Route Handlers and Server Actions). 
* **Server Components by Default:** First default to React Server Components (RSC) for data fetching. Introduce `"use client"` only for interactivity, hooks, or Chart.js visualizations.
* **Data Fetching:** Fetch data server-side via Prisma in Server Components OR client-side fetching to the Next.js `/api/...` route handlers. Use Route Handlers for webhooks.

## 2. Authentication (Clerk)
* **No Custom Auth Logic:** We use Clerk for all authentication and identity management. Never generate code for custom password hashing, JWT generation, or session cookies.
* **Server-Side Auth:** When verifying users in Server Components, Route Handlers, or Server Actions, use `auth()` or `currentUser()` from `@clerk/nextjs/server`. 
* **Client-Side Auth:** When UI components need auth state, use the `useUser()` or `useAuth()` hooks from `@clerk/nextjs`.

## 3. Database & Prisma ORM
* **Prisma Singleton:** Always import the Prisma client from a centralized singleton file (e.g., `lib/prisma.ts`) to prevent connection exhaustion during hot-reloading in development. Never instantiate `new PrismaClient()` directly inside a component or route.
* **Relational Logic:** Postgres uses the `clerk_id` to link Clerk identities to the application's relational data.

## 4. TypeScript & Syntax Rules
* **Strict Typing:** Never use `any`. Define interfaces or type aliases for all frontend component props, API request bodies, and API responses. 
* **Formatting:** When outputting code blocks in markdown, always use the `ts` or `tsx` tag appropriately.

## 5. UI & Styling (Tailwind + ShadCN)
* **Design System First:** Base UI code strictly on UI conventions described in `.agent/design.md`.
* **ShadCN Integration:** Assume ShadCN components are available in `@/components/ui/`. Leverage these primitives to save time instead of building raw HTML inputs and dialogs from scratch.
* **Chart.js:** Build all visual graphs using Chart.js or react-chartjs-2. Do not use Recharts as this project mandates Chart.js.
